    • Case Study: Azure Worker Service for Real-Time Data Processing

    How Intellectix built a real-time processing service to sync data between three systems.

    Project Description:

    The EOIR Courts & Appeals System (ECAS) initiative is part of an overarching information technology modernization effort at the Executive Office for Immigration Review (EOIR). The goal of ECAS is to phase out paper filing and processing, and to retain all records and case-related documents in electronic format. ECAS is comprised of specialized applications and tools. In support of the EOIR mission, it will further enable the timely, fair, and uniform adjudication of immigration cases.

    Challenge

    Microsoft Dynamics was selected as the platform for a new application to support Immigration Judges’ ability to access, manipulate, and annotate electronic documents during immigration hearings. The Dynamics platform as implemented by another vendor required local copies of judicial hearing and document data in order to support the intended functionality. This data was sourced from over 40 tables located across multiple databases. Furthermore, the data needed to be accurately passed to the underlying Dynamics database in less than one minute in order to ensure timely document interactions.


    Solution

    Intellectix, in partnership with NTT Data and Microsoft, developed and implemented a worker service as an Azure Function App to queue and process all changes committed to each of the disparate databases. The worker service relied on CDC (change data capture) SQL tables for awareness of data which needed to be passed to Dynamics. The worker then used the primary keys of the changed rows to read the relevant data from the source table. This solution prevented contention issues which might have otherwise been presented by asynchronous processing. Additionally, the team built an administrative console with a dashboard and an ability to monitor and manually re-process individual rows as needed.


    Results

    • Reduced processing time of judicial hearing and document data from several hours to under 1 minute by using Azure Function Apps
    • Provided infinite scalability at a throughput of more than 9.4 records/sec per worker from multiple SQL databases to Dynamics through the API
    • Enabled real-time reporting capability to monitor overall health of the workers and check the status of any given row
